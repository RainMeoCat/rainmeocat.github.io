【論文閱讀筆記】Video Instance Segmentation Tracking with a Modified VAE Architecture

Abstract 摘要

作者提出了一種建立於Mask R-CNN的自動編碼器架構，用於video instance segmentation，該架構使用一個共享編碼器與三個分支出來的解碼器。為此作者引入一個高斯過程(Gaussian process)來放寬先驗條件來增強VAE的統計表示，三個分支分別用於：

  • 預測未來幀
  • 產生物體外框
  • 影片物件分割掩碼

此架構能夠學習影片資料集中的空間依賴性與連續運動，建立高品質、準確的分割掩碼。

1 Introduction 簡介

VIST(Video Instance Segmentation Tracking，VIST)目的是對影片中的所有物件實例進行分類、定位、分割和追蹤，能夠對於場景有更加詳細的理解，能夠用於自動駕駛與影片自動編輯。

VIST與傳統物件追蹤不同的是在遮擋嚴重的場景，簡單的矩形邊界框無法正確表示物體的形體，VIST能夠產生二進位制分割遮罩和像素級的追蹤結果，此外，物體被遮擋，或者在重新出現之前離開場景幾幀，實例(物體)的身份會被保持到重新出現時。

VIST在半監督模式下，感興趣(標記)的物件的初始掩碼是在第一幀中就提供的；在非監督模式中，只追蹤突出的物件。

VIST能隨著遮擋保留實例身份的特性牽涉到時間訊息，在CNNs時代時應用VIST於現實環境仍具有挑戰性，為此催生了：

  • 納入了時間訊息，並增加了一個關聯標頭來連結物件身份隨時間的變化的Mask R-CNN
  • MaskTrack R-CNN，引入新的追蹤分支，共同完成檢測、分割和追蹤任務。

然而，這兩種方法都假設Mask R-CNN能夠有效地產生定位良好的邊界框和準確的分割結果。在高度多樣化和非結構化的影片中，視覺對象往往會受到部分甚至全部的遮擋、變形、姿勢變化的影響，且很多情況下，物件的外觀相似，很難從雜亂的背景中分離出來。

因此，物體檢測極有可能出現初始化不良的情況，進而降低邊界框內預測的物體掩碼的精度。

使用關聯標頭直接跨幀連結Mask R-CNN分割掩碼的機制面臨著固有的侷限性：該模型在處理假陰性假設(false negative proposals)方面存在困難，導致效能較差。

作者注意到：跨幀的空間依賴性和運動連續性提供了一個支援性的上下文，使影片模型能夠更好地推斷下一步發生的事情。

作者利用這個上下文特性，提出了一種建立在Mask R-CNN之上的變異自動編碼器（VAE），能捕獲所有實例共享的空間和運動訊息，並產生線索以輔助學習，減少虛假的負面掩碼預測。

透過強迫網路解決多個學習任務，作者在誘匯出一個特徵，該特徵(線索)能完美地引導影片實例分割追蹤任務，並產生高品質的分割掩碼。圖1展示了作者的方法在MOTS和YouTube-VIS資料集上的實驗結果。

貢獻總結：

  • 其多工網路架構和訓練方案經過設計，充分利用了空間和運動線索。在最近發佈的KITTI MOTS、MOTS Challenge和YouTube-VIS資料集上實現了此網路新的效能。
  • 引入了高斯過程模型，透過放寬傳統VAE的先驗強獨立和同分佈(iidiidiid)假設，並提取出潛變數之間存在的相關性，來增強VAE的統計表示。

2 Related Work 相關工作

影像實例分割（Image Instance Segmentation）

實例分割任務，與物件檢測和語義分割密切相關。解決這一任務的主流架構是在網路中增加一個分支來預測邊界框或區域建議內的物體掩碼。
Mask R-CNN，用掩碼標頭擴充了Faster R-CNN框架，並達到了最佳的效能。
PANet和Mask Scoring R-CNN進一步發展了這一思想，它們在COCO資料集上的表現優於競爭方法。
這些方法在影像中的像素級定位感興趣的物件上取得了令人印象深刻的效能。
但它們並不能直接適用於VIST任務。

在VIST任務中，物體實例不僅在每一幀影片中被分割並由掩碼表示，而且在整個影片中以
相同的對應身份進行追蹤。

視覺物件追蹤

典型的視覺物件追蹤（Visual Object Tracking）是基於邊界框的，通常不能提供精確的物件輪廓。
VOT、MOT Challenge、PETS和KITTI是用於評估VOT效能的流行資料集。

現有的該任務方法大致可分為兩類，即免檢測和基於檢測的方法。

  • 免檢測追蹤方法
    在第一幀中給定一個固定數量的物件進行手動初始化，追蹤物件。
    它們不能處理影片序列中間出現的新物體。
  • 基於檢測的追蹤
    一般要求檢測到物體後，由追蹤器將檢測區域連線起來，形成目標的軌跡。為瞭解決物體檢測連結的模糊性，許多努力都用資料關聯的方法來探索這個問題，如馬爾科夫決策過程、事件聚合、貪婪演演算法和注意力相關過濾器等。

影片物件分割

影片物件分割（Video object segmentation）旨在分割和追蹤影片中的物件，但不需要識別物件類別。VOS任務的流行資料集包括DAVIS、SegTrackV2和YouTube-VOS。一般來說，VOS有兩大類：半監督和無監督。

半監督方案

在第一個掩碼中給出了真實/正確掩碼(ground-truth masks)，並透過影片序列的餘下幀進行追蹤。通常，研究的是基於空間-時間圖和CNN的方法。空間-時間圖方法依賴於兩個重要線索。

  1. 圖結構的物件表示，如像素、超像素、物件補丁
  2. 時空推理(spatial-temporal reasoning)，如時空網格(spatial-temporal lattices)、最近鄰域和樹的混合模型。

一些基於CNN的方法採用具有光流法的循環神經網路(Recurrent Neural Networks)來捕捉物體運動的時間一致性，並在幀之間傳播訊息。另一種方法則將VOS表述為一個像素匹配問題。而其他方法傾向於學習一個外觀模型來執行每一幀物體的畫素級檢測和分割。

無監督方案

任務是以全自動的方式分割突出的前景移動物體。運動模式（如光流法和長期軌跡）通常被用作主要的訊息來源。

由於缺乏物體遮罩的引導，如果不同實例和動態背景之間存在運動模糊性，大多數無監督方法無法分割特定的物體。

綜上所述，VIST與VOT和VOS有一些共同的挑戰，但在幾個方面有所不同，例如：

  • 沒有給出第一個掩碼作為引導
  • 需要對屬於一組預先定義類別的所有物件實例進行分類、分割，並在整個影片中以一致的分配身份進行追蹤
  • 最終輸出的是精確的畫素級掩碼
  • 掩碼之間沒有任何重疊畫素
  • 為了評估VIST的效能，需要對一個預先定義類別別集的所有實例的掩碼和跨幀的實例標識進行標註。

得益於最近引入的KITTI MOTS、MOTS Challenge和YouTube-VIS資料集，可以評估所提出方法的有效性。

3 Method 方法

3.1 Overview 概述

作者採用VAE架構，它由一個概率變數來描述潛伏空間中的觀測值。作者的方法如圖2所示。具體來說，作者的VAE架構包括一個編碼器和三個解碼器，從而產生三個並行分支，即輔助分支、提案分支和增強分支。

在提案分支和增強分支中，降採樣編碼器和升採樣解碼器之間採用跳轉連線方案，以達到訊息儲存的目的。這三個分支共享相同的隱藏特徵層，但執行不同的任務。

輔助分支(auxiliary branch)
接受幀級影片輸入，並學習預測未來幀。該分支的目標是引導網路學習更精細的表徵，增加編碼在潛伏空間中的有意義的語義訊息量。

提議分支(proposal branch)
總結並輸出物件級訊息，用於連線物件的時間。它還提供線索，以減少增強分支中的假否定。

增強分支(augment branch)
聚合了從VAE編碼器和Mask R-CNN網路中不同層提取的像素級特徵。低級特徵包含豐富的空間細節，高級特徵則包含更多的語義訊息。透過將這些提取的特徵與提案分支和Mask R-CNN網路的輸出相結合，該分支產生最終的實例分類、身份、檢測框和分割掩模。

3.3 Decoder Branches 解碼器分支

3.3.1 Auxiliary Branch 輔助分支

在VAE中，如果網路設計不當，編碼器和解碼器之間可能存在繞過的連線。具體來說，**如果解碼器可以直接和確定地訪問源，潛在變數zzz可能不會捕捉到很多訊息，這樣VAE就不能在這個過程中發揮有效作用。**為了避免這種情況，作者加入了一個輔助分支，作為一個支援性的角色，引導模型增加潛空間中編碼的語義訊息量，並建立訊息豐富的潛變數。

該分支的訓練目標是重建給定當前觀測值ξtξ_tξt​的未來幀Ft+1F_{t+1}Ft+1​。重建損失由預測影像和地面真實影像之間的均方誤差（MSE）來衡量。輔助分支用n個殘差塊來實現，遵循中提出的架構，不進行批量歸一化。作者在每個殘差塊之後使用步長為2的步長摺積對輸入進行下取樣，直到一個瓶頸層，作者利用亞畫素摺積在兩個連續的殘差塊之間進行上取樣。所有摺積層都由3x3濾波器組成。以下兩個分支的架構類似。

4 Experiments and Analysis 實驗與分析

4.1 Experimental Setup 實驗步驟

資料集。作者在新引入的KITTI MOTS、MOTSChallenge和YouTube-VIS資料集上評估了所提出的方法，以進行影片實例分割追蹤。這些資料集中的物件在各幀中具有一致的實例身份標簽。

  • KITTI MOTS資料集專注於來自車載攝像頭的影片。它包含21個場景中的8,008個幀，有26,899個注釋的汽車和11,420個注釋的行人。
  • MOTS挑戰資料集展示了擁擠場景中的行人。它包含2,862幀，有26,894個注釋的行人。
  • YouTubeVIS資料集包含網際網路影片，涵蓋40個類別，如動物、汽車配件和人類。

然而，釋出的測試和驗證集中的注釋並不包括物件實例身份，因此不適合訓練和測試VIST方法。因此，作者將YouTube-VIS訓練集隨機分成2038個訓練影片和200個測試影片。

這些資料集有一些區別：KITTI MOTS和MOTSChallenge資料集包含較少的類，但有更長的影片，有更多的物體經常在場景中消失和重新出現。

評價指標。按照TrackR-CNN，作者報告了評價指標：

  1. 軟多目標追蹤和分割精度（sMOTSA）
  2. 多目標追蹤和分割精度（MOTSA）
  3. 多目標追蹤和分割精度（MOTSP）

作者還報告了餘下指標：真陽性（TP）、假陽性（FP）和假陰性（FN）。(用於表1)

在這些指標中，sMOTSA是推薦用於衡量效能的主要指標，因為它考慮了分割以及檢測和追蹤品質。

實施細節。作者使用ResNet-101作為Mask R-CNN的主幹，並在COCO和Mapillary上對其進行預訓練，用於KITTI MOTS和MOTSChallenge資料集的實驗，並在COCO上對YouTube-VIS資料集的實驗進行預訓練。作者在PyTorch中實現作者的模型，並用4個GeForce RTX 2080 Ti GPU進行訓練。每個批次有8張圖片（每個GPU持有2張圖片）。作者使用Adam優化器在目標資料集上訓練模型20個歷時，學習率為1.5×10-4。

作者使用不同類別的加權實例掩碼產生影像級別的組合分割掩碼，並與影像串聯作為編碼器的輸入。影像的長邊和短邊分別被調整為1333和800像素，不改變長寬比。

作者對所提出的方法與MaskTrack R-CNN和TrackR-CNN進行了全面的比較，並在表1中報告了影片實例的分割和追蹤效能。Mask RCNN+IT表示透過IoUTracker的IoU匹配來連線MaskR-CNN輸出掩碼的基本基線。為了進行公平的比較，作者在作者的硬體平台上重新訓練MaskTrack R-CNN和TrackR-CNN，使用相同的ResNet-101主幹並遵循他們的訓練和評估協議。MaskTrack R-CNN和TrackR-CNN之間的一個重要區別是，MaskTrack R-CNN允許分割掩碼重疊，而TrackR-CNN要求沒有重疊的掩碼。在現實中，每個畫素應該只屬於一個物體。因此，在VIST任務中，作者定義面具之間不能有任何重疊的畫素；每個畫素只分配給一個物件實例。

5 Conclusion 結論

作者引入了一個統一的變異自動編碼器修改，用於可靠地分割和追蹤不同的非結構化影片中的多個實例，這些影片中存在大量的物體遮擋和變形，影響了許多現有方法的執行方式。在幾個不同的資料集上的實驗證明了所提出的方法與最先進的方法相比的優越性，這些方法透過增加一個關聯模組來追蹤物體的時間。
